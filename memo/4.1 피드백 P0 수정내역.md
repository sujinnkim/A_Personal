# 4.1 제출 피드백 — P0 수정 내역 (최종: QA 정렬)

> 대상: [4.1 제출 피드백 및 수정포인트.md]의 **P0 (이슈/목표/QA/드라이버/전략 분해 단위 불일치)**
> 최종 방향: **아키텍처 드라이버(AD)를 품질 요구사항(QA)에 정렬.** 이슈:AD는 다대일 허용.
> 적용일: 2026-07-05 · docs(원본)·report(파생) 동시 반영

## 0. 수정 경위 (중요)

처음엔 평가자의 "이슈:AD 1:1" 지적을 문자대로 따라 **13개 원인 드라이버**로 재편했으나, 문서 자체 골격(QA·DG 추적)에 대조한 결과 다음 문제가 드러나 **QA 정렬 10개로 정정**함:
- 드라이버 altitude 불일치(가용성=속성 수준 vs lock경합=메커니즘 수준)
- QA 추적 오류(외부지연 드라이버→QA-01, lock 드라이버→QA-03 등 잘못된 연결)
- 간판 품질속성 "가용성(성공률 99.9%)"을 드라이버에서 강등

정통 원칙(ATAM/ADD): **드라이버는 유의미한 요구(QA)에 정렬되고, 이슈↔QA는 원래 다대다.** 평가자가 본 "다대다"는 대부분 정상적인 관계였고, 진짜 결함은 아래 3가지뿐이었다.

## 1. 실제 결함과 최소 수정

| 결함 | 수정 |
| ----- | ----- |
| 옛 AD-04가 가용성(QA-04)과 외부 서버 장애 격리(QA-05) **두 속성을 뭉갬** | AD-04(가용성) + AD-05(외부 서버 장애 격리) **분할** → 평가자 "QA-05↔04 중복" 지적을 정통 방식(sibling 드라이버)으로 해소 |
| AD 설명칸에 설계전략(해법) 노출 | 설명을 **근본 원인 중심**으로 재작성 |
| ISSUE-07(lock)이 커넥션풀 드라이버(옛 AD-03)에 **오배치** | ISSUE-07 → AD-04(가용성)로 정정 |

추가로 옛 AD-04가 담던 QA-05를 AD-05로 분리하면서, 기능 드라이버는 +1 시프트(옛 AD-05·06·07 → AD-06·07·08), 제약 드라이버는 번호 유지(AD-09·10).

## 4. QA 층위 (5개 유지)

QA-04(핵심 기능 가용성)는 다른 QA가 함께 충족될 때 달성되는 **종합 결과 지표**, QA-01·02·03·05는 각각 **독립 원인·메커니즘 기준**. 특히 QA-03(내부 커넥션풀 격리)·QA-05(외부 서버 장애 격리)는 서로 다른 결함 격리 축. → 표36(저해 요인)은 "각 요인은 QA-01/02/03/05에서 개별로 다룸"으로 재프레이밍.
