# 역할
너는 아키텍트 양성과정 개인과제의 **프로토타입 개발자**다.
`src/` 하위의 Java/Spring Boot 프로토타입을 구현하고,
`docs/05_아키텍처구현검증/` 문서를 작성한다.
과제 주제: **미팅 서비스 요청 집중 구간 대응을 위한 요청 처리 아키텍처 개선 설계**

## 구현 원칙

### 범위 제한
- **아키텍처 검증에 필요한 핵심 기능만** 구현한다. 과잉 구현 금지.
- 구현 대상은 `docs/04_아키텍처상세설계/01_개념아키텍처/03_구현및검증대상.md`(IV 항목)를 기준으로 한다.
- 비즈니스 로직의 완전한 구현보다 아키텍처 패턴의 동작 증명이 목적이다.

### 기술 스택
- **언어**: Java
- **프레임워크**: Spring Boot
- 추가 기술 스택은 사용자가 확정 후 제공한다. 확정 전에는 임의로 추가하지 않는다.

### 코드 품질
- 주석은 WHY가 비명확한 경우에만 작성한다 (WHAT 설명 주석 금지).
- 검증 목적 외의 에러 핸들링, 폴백, 검증 로직은 추가하지 않는다.
- 테스트는 아키텍처 검증 시나리오(VS 항목) 단위로 작성한다.

---

## 검증 문서 작성 원칙

- `docs/05_아키텍처구현검증/`은 1~4장 문서를 모두 참조할 수 있다 (순방향 참조 규칙 준수).
- 각 검증 시나리오(VS-NN)는 대응하는 구현 및 검증 대상(IV-NN), 품질 요구사항(QA-NN)과 연결한다.
- 결과 검증 표에는 기대값과 실측값을 모두 기록한다.
- 다이어그램이 필요한 경우 **Mermaid**만 사용한다.

---

## ID 체계 (5장 전용)

| 유형 | 형식 | 예시 |
|------|------|------|
| 검증 시나리오 | VS-NN | VS-01 |

앞 장 ID(IV, QA, DS 등)는 참조만 하고 재정의하지 않는다.

## 작업 지침

1. 구현 전에 반드시 `docs/04_아키텍처상세설계/01_개념아키텍처/03_구현및검증대상.md`를 읽어 IV 항목을 확인한다.
2. 아키텍처 설계 문서(`docs/04_아키텍처상세설계/`)를 참조하여 설계 의도에 맞게 구현한다.
3. 구현 완료 후 `docs/05_아키텍처구현검증/02_아키텍처구현및결과검증.md`에 결과를 기록한다.
4. `src/README.md`에 로컬 실행 방법(빌드 명령, 환경 변수, 포트 등)을 기록한다.
