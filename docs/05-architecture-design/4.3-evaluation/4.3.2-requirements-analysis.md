# 4.3.2. 요구사항 관점의 분석

본 절은 설계(4.1·4.2)와 결정표(4.3.1)가 *상위 요구사항을 빠짐없이 반영하는가*를 추적한다. 추적 대상은 3장 요구사항 정의의 두 계층으로, (가) **기능 요구사항**(2.1 UC-01~08)과 (나) **품질 요구사항**(2.2 QA-01~05)이며, 각 요구가 어느 설계 결정(ADR)·모듈로 반영되었는지 명시한다. 기술 스택·API 호환성 등 *구조 제약*(CR)은 요구가 아닌 전제이므로 4.3.1 결정표에서 결정과 함께 추적한다.

> 본 절은 설계가 요구를 *반영하는가*(커버리지)만 평가한다. 각 요구가 부하·장애 환경에서 실제로 충족되는지를 측정하는 검증 시나리오·깊이는 별도의 검증 활동 소관이며 본 설계 문서의 범위가 아니다.

## (가) 기능 요구사항 (UC): 실현 추적

기능 요구사항의 실현 책임은 4.2.2 모듈 뷰가 각 UC를 특정 도메인 모듈에 배정함으로써 진다.

| ID | 유즈케이스 | 실현 위치 (모듈 / 관련 ADR) |
|---|---|---|
| **UC-01** | 사용자 권한 갱신 | `domain.auth`(AuthService): ADR-003 캐시 + ADR-005 선제 적재 + ADR-009 CB fallback |
| **UC-02** | 회의 예약 | `domain.meeting` 예약 등록: 예약 데이터가 ADR-005·006 피크 예측의 입력이 됨 |
| **UC-03** | 회의 시작 | `domain.meeting`(MeetingService) + `integration.meetingManager`(회의 개설, 뒤단 VC/AC): ADR-002 비동기 + ADR-009 CB |
| **UC-04** | 회의 입장 | `domain.entry`(MeetingJoinService): ADR-002 비동기 + ADR-004 전용 Connector + ADR-008 Bulkhead |
| **UC-05** | 회의 퇴장 | `domain.meeting` 참석자 상태 변경(cPaaS 피드백): ADR-007 Command/Query 경로 분리 |
| **UC-06** | 참석자 초대 | `domain.meeting`: ADR-008 service-pool 격리 |
| **UC-07** | 회의 종료 | `domain.meeting` + `integration.meetingManager`(회의 종료): ADR-009 CB fallback |
| **UC-08** | 알림 발송 | `domain.meeting` 통보 경로 |

→ 기능 요구사항 **UC-01~08 전 8항이 모듈 뷰에 배정**되어 실현이 확정되며, 미반영 요구는 없다.

## (나) 품질 요구사항 (QA): 정량 임계 정렬

| ID | 품질 요구사항 (정량 임계) | 반영 내용 (ADR / AS) |
|---|---|---|
| **QA-01** | 피크 시 권한 갱신 평균 응답 **1초 이내** | ADR-003(L1+L2 캐시)로 외부 호출 완충 + ADR-005(선제 적재)로 cold start 제거 + ADR-009(CB fallback)로 외부 지연 차단 |
| **QA-02** | 8만 동시 입장 시 커넥션 풀 **80% 이하·응답 1초 이내** | ADR-002(비동기)로 서블릿 스레드 즉시 반환 + ADR-004(전용 Connector)로 우선순위 보장 + ADR-008(Bulkhead)로 커넥션 격리 + ADR-005(선제 적재)로 피크 초입 입장 cold start 완충 |
| **QA-03** | 입장 풀 고갈 시 회의 시작 등 타 기능 **성공률 100%** | ADR-001(도메인 경계)로 귀속 기준 마련 + ADR-008(기능별 풀 분리)로 물리 격리 |
| **QA-04** | 트래픽 집중 구간 핵심 기능 **성공률 99.9%** | 복합: ADR-002·004·006·008이 입장 경로를 다층 보호, ADR-005이 피크 초입 완충 |
| **QA-05** | 외부 서버 장애 시 무관 기능 **성공률 99.9%** | ADR-009(차등 CB + 계층 Fallback)로 장애 서버 격리 + ADR-008(스레드 Bulkhead)로 전파 차단 |

## 요구사항 충족도 종합

| 구분 | 항목 수 | 반영 |
|---|:--:|:--:|
| 기능 요구사항 (UC) | 8 | 8 (100%) |
| 품질 요구사항 (QA) | 5 | 5 (100%) |

**평가.** 상위 요구사항 13항(UC 8 + QA 5)이 모두 설계에 추적되며 미반영 요구는 없다. 기능 요구사항은 모듈 뷰에 배정되고, 품질 요구사항은 설계 결정(ADR/AS)에 1:1 또는 다대일로 매핑된다. 다만 커버리지가 확인되었다고 실제 임계 달성이 보장되는 것은 아니므로, 각 요구가 부하·장애 환경에서 정량 임계를 충족하는지는 실측 검증으로 확인해야 하며 그 시나리오·깊이는 본 절의 범위 밖이다.
