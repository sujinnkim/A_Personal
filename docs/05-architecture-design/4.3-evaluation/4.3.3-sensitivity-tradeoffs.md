# 4.3.3. 민감점과 상충점

본 절은 ATAM 관점에서 **민감점**(Sensitivity Point, 한 설계 파라미터 값이 특정 품질 속성을 강하게 좌우하는 지점)과 **상충점**(Tradeoff Point, 한 결정이 둘 이상의 품질 속성을 반대 방향으로 끌어당기는 지점)을 식별한다. 3.2절 각 AS의 §채택안의 한계와 완화에서 도출된 트레이드오프를 집계하며, 품질 속성 표기는 3.1의 AD-01~AD-05를 따른다(AD-01 권한 갱신 응답시간, AD-02 동시 입장 처리 성능, AD-03 커넥션 풀 장애 격리, AD-04 핵심 기능 가용성, AD-05 외부 서버 장애 격리).

## 민감점 (Sensitivity Points)

| ID | 민감점 (파라미터) | 영향 품질 속성 | 현재 설계 결정 · 완화 |
|----|------------------|--------------|---------------------|
| **SP-01** | join-pool 크기 (100) | AD-02 응답성, AD-03 격리 | 작으면 8만 동시 입장에서 커넥션 획득 타임아웃, 크면 DB 총량(CR-02) 압박. 세션 한계 안에서 부하 시험으로 확정 (AS-08) |
| **SP-02** | externalCallExecutor 크기 (core 100 / max 500) | AD-02 응답성 | 작으면 외부 호출 대기·거부↑, 크면 스레드 자원 소비↑. CB로 외부 지연 조기 차단 후 부하 시험 확정 (AS-02) |
| **SP-03** | 캐시 TTL (L1 5분 / L2 30분~1h) | AD-01 응답성, AD-04 정합성 | 길면 hit율↑·응답성↑이나 권한 stale↑. 저빈도 데이터 한정 + 변경 이벤트 `@CacheEvict` 즉시 무효화 (AS-03) |
| **SP-04** | 서버별 CB 임계값 (failureRate · waitDuration) | AD-05 장애 격리 | 낮으면 정상 서버 과도 차단, 높으면 장애 감지 지연. 서버 특성 기반 초기값(AS-09 표 56)을 부하 시험으로 보정 (AS-09) |
| **SP-05** | Throttling 임계 RPS (비핵심 API) | AD-02 응답성, AD-04 가용성 | 낮으면 비핵심 UX 저하, 높으면 핵심 경로 여유 부족. 피크 예측 구간·비핵심 API 한정 후 모니터링 보정 (AS-06) |

## 상충점 (Tradeoff Points)

| ID | 상충점 (결정) | 관련 품질 속성 (상충 방향) | 본 설계의 절충 결정 |
|----|--------------|--------------------------|-------------------|
| **TP-01** | 캐시 TTL 설정 (AS-03) | AD-01 응답성 ↔ AD-04 정합성 | TTL↑이면 외부 호출↓·응답성↑이나 권한 반영 지연. → 저빈도 권한 데이터로 한정하고 변경 이벤트 시 `@CacheEvict`로 stale 최소화 |
| **TP-02** | 기능별 풀 분리 (AS-08) | AD-03·AD-04 신뢰성(격리) ↔ 자원 효율(CR-02 총량) | 격리↑이나 정적 분할로 총 커넥션↑. → 풀 합을 DB 상한 내로 설계해 격리·총량을 동시 만족 |
| **TP-03** | 외부 호출 비동기 전환 (AS-02) | AD-02 응답성 ↔ 유지보수성(복잡도) | 처리량↑·스레드 고갈 방지이나 컨텍스트 전파·예외 처리 복잡. → TaskDecorator·fallback 표준화로 복잡도 흡수 |
| **TP-04** | Primary/Replica 라우팅 (AS-07) | AD-02 응답성 ↔ 정합성(복제 지연) | read를 Replica로 분산하나 read-after-write 불일치. → 강정합 불필요 조회만 Replica, 입장 직후 재조회는 Primary |
| **TP-05** | 서버별 차등 CB (AS-09) | AD-05 신뢰성 ↔ 운영성(튜닝 부담) | 정밀 격리이나 서버별 파라미터 튜닝·운영 복잡도↑. → 특성 기반 초기값으로 튜닝 출발점을 명확히 한다 |
| **TP-06** | 예약 기반 선제 초기화 (AS-05) | AD-04 응답성(cold start 제거) ↔ 자원 효율 | 피크 초입 응답성↑이나 예측 오차 시 불필요한 외부 호출. → 임계값 이상 회의로 워밍 대상 한정 + 배치·딜레이 분산 |

## 핵심 상충 해소 방향

본 아키텍처의 상충점은 대부분 **AD-02 시간 응답성**을 한 축으로 한다. 캐시 TTL·비동기 전환·Replica 라우팅·풀 분리가 모두 "응답시간(1초 임계)을 지키기 위해 다른 품질(정합성·자원·유지보수성)을 어디까지 내줄 것인가"를 묻는다. 본 설계는 *응답시간 임계를 불변으로 고정*하고 나머지 품질을 메커니즘(적용 범위 한정·무효화 이벤트·총량 상한·표준 초기값)으로 보완하는 방향으로 일관되게 절충했다. 따라서 이 절충들이 실제로 유효한지의 확인은 대부분 부하 시험(실측)에 집중되며, 정합성 절충(TP-01·04)은 사건·무효화 주입으로 확인한다. 절충에서 남은 잔여 위험은 4.3.4로 넘긴다.
