# 4.1.3. 구현 및 검증 대상

전체 설계 범위와 프로토타입에서 구현·검증하는 범위를 구분하고, 각 검증 항목의 달성 기준과 방법을 정의한다.

---

## 다이어그램 범례

4.1.1 개념도에서 사용하는 표기를 정의한다.

| 표기 | 의미 |
|-----|-----|
| `subgraph 이름["레이블"]` | 아키텍처 경계 · 레이어 |
| 노드 레이블의 `AS-XX` 접두사 | 해당 컴포넌트에 적용된 설계전략 |
| `-->` 실선 화살표 | 주 요청 · 응답 흐름 |
| `--->` 긴 실선 화살표 | 캐시 조회 흐름 |
| `-.->` 점선 화살표 | 조건부 · 선택적 흐름 |
| `style ... fill:#ff9999` | 오류 · 고갈 상태 노드 |

## 검증 항목 (IV: Implementation Verification)

각 IV는 아키텍처 품질 요구사항(QA)의 달성 여부를 프로토타입으로 검증한다.

| IV | 검증 목표 | 관련 QA | 관련 AS | 검증 방법 | 프로토타입 제약 |
|----|---------|--------|--------|---------|-------------|
| IV-01 | 동시 입장 집중 구간에서 DB 커넥션 풀 사용률 80% 이하, 평균 응답시간 1초 이내 | QA-02 | AS-02, AS-04, AS-08 | k6 부하 · HikariCP activeConnections 메트릭 · 응답시간 분포 | C-06: 8만 명 축소 시뮬레이션 (단일 서버) |
| IV-02 | join-pool 고갈 시 service-pool 격리: 회의 조회 API 성공률 100% 유지 | QA-03 | AS-08 | join-pool 의도적 포화 후 조회 API 동시 호출 · 성공률 확인 | — |
| IV-03 | 피크 시 권한 갱신 응답시간 1초 이내: 캐시 hit율 90% 이상 | QA-01 | AS-03, AS-05 | k6 로그인 부하 · L1/L2 캐시 hit율 · 외부 호출 횟수 비교 | — |
| IV-04 | 외부 서버 장애 주입 시 무관 핵심 기능 성공률 99.9% 이상 | QA-05 | AS-09 | Stub 장애 주입 (응답 지연 / 오류) 후 입장 · 조회 API 성공률 측정 | C-05: 외부 서버 Stub 대체 |
| IV-05 | write 집중 중 참석자 상태 조회(read) 성공률 99.9% 이상·응답 유지 | QA-04 | AS-07 | 피드백 write 집중(Primary) + 상태 조회 read(Replica) 동시 부하 · 라우팅 분리 확인 | — |

> **QA-04(가용성)는 종합 지표다.** 격리(AS-08·09)·비동기(AS-02)·전용 경로(AS-04)·cold start 완충(AS-05)·유입 제한(AS-06)과 배치 뷰(4.2.3)의 이중화·failover가 합성되어 달성되며, 단일 전담 전략을 두지 않는다. 검증도 특정 IV 하나가 아니라 핵심 기능 성공률을 측정하는 IV-01·IV-04·IV-05의 합으로 확인하고, IV-05는 그중 write/read 경합(ISSUE-07) 국면을 담당한다.
