# 4.2.1.3. UC-01 권한 갱신: 캐시 hit/miss (AS-03·AS-02·AS-09)

로그인 후 권한 갱신 시 L1·L2 캐시 분기 흐름이다. 캐시 적중 경로와 L2 miss 시 외부 권한 병렬 조회 경로로 나눈다. Overall View의 F1·D2·C1·C2·I2·I3 구간을 확대한다.

## (1) 캐시 적중 경로

```mermaid
sequenceDiagram
    participant C as 클라이언트
    participant E as 일반 Connector
    participant F as ThrottlingInterceptor
    participant D as domain.auth
    participant L1 as L1 Caffeine
    participant L2 as L2 Redis

    C->>E: ① GET /members/{email}
    E->>F: ② 처리량 확인
    F->>D: ③ 통과 허용
    D->>L1: ④ L1 조회
    alt L1 hit
        L1-->>C: 권한 즉시 반환
    else L1 miss
        L1-->>D: miss
        D->>L2: ⑤ L2 조회
        alt L2 hit
            L2-->>D: 권한 반환
            D->>L1: L1 적재
            D-->>C: 응답
        else L2 miss
            L2-->>D: miss (아래 (2)로 이어짐)
        end
    end
```

## (2) L2 miss 시 외부 권한 병렬 조회·적재

```mermaid
sequenceDiagram
    participant D as domain.auth
    participant Iac as integration.ac
    participant Icp as integration.copilot
    participant AC as AC서버
    participant CP as Copilot Admin
    participant Cache as 캐시(L1·L2)

    par AC 권한
        D->>Iac: ⑥ AC 권한 조회
        Iac->>AC: ⑦ 비동기 호출
        AC-->>Iac: AC 권한
    and Copilot 권한
        D->>Icp: ⑥ Copilot 권한 조회
        Icp->>CP: ⑦ 비동기 호출
        CP-->>Icp: Copilot 권한
    end
    D->>Cache: ⑧ L1·L2 동시 적재
    D-->>D: ⑨ 권한 응답 반환
```

## AS 적용 지점 요약

| 스텝 | 지점 | 적용 AS | 효과 |
|:---:|---|:---:|---|
| ② | ThrottlingInterceptor | AS-06 | 피크 구간 비핵심 API 처리량 제한(권한 갱신에 상한 적용) |
| ④ | L1 Caffeine 조회 | AS-03 | 인스턴스 로컬 hit로 네트워크 없이 즉시 반환 |
| ⑤ | L2 Redis 조회 | AS-03 | 인스턴스 간 공유 캐시로 외부 중복 호출 방지 |
| ⑦ | externalCallExecutor 비동기 병렬 | AS-02+AS-09 | AC·Copilot CompletableFuture 병렬 조회, CB 보호 |
| ⑧ | L1·L2 동시 적재 | AS-03 | miss 후 외부 결과를 양 계층에 적재해 후속 hit 보장 |
