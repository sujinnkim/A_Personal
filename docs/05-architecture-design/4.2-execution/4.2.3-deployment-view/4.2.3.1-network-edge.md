# 4.2.3.1. 네트워크·엣지

운영 환경은 전용 VPC 위에 구성한다. 인바운드는 `Cloud HTTPS LB → Nginx Ingress → front-api`, 외부 서버 연계(아웃바운드)는 PSC/Cloud VPN으로 한정한다.

## VPC·서브넷

| 서브넷 | 용도 | 접근 |
|---|---|---|
| `gke-subnet` | GKE 노드·Pod (front/server/admin/edge/observability) | LB·내부만 |
| `data-subnet` | Cloud SQL·Cloud Memorystore (Private Service Access) | GKE에서만 사설 IP |
| `psc-subnet` | 외부 서버 연계용 PSC/VPN 엔드포인트 | 아웃바운드 한정 |

## 방화벽·격리

- **인바운드**: Cloud HTTPS LB → GKE 인그레스만 허용, 그 외 노드 직접 접근 차단.
- **데이터 접근**: Cloud SQL·Memorystore는 Private Service Access로 GKE에서만 사설 IP 접근. 외부 공인 IP 미노출.
- **NetworkPolicy**: 네임스페이스 간 통신을 필요한 경로만 허용(edge→front/server/admin, front/server/admin→data). admin 네임스페이스는 외부 인입 차단.

## 엣지: Cloud HTTPS LB + Cloud Armor

| 항목 | 결정 |
|---|---|
| 노출 | Cloud HTTPS LB만 공인, GKE는 private |
| 보안 | Cloud Armor(WAF·rate limit·IP 정책)를 LB 앞단에 적용 |
| 대상 | Nginx Ingress(edge 네임스페이스) |

## Nginx 라우팅 (AS-04)

`nginx.conf` 기준으로 URL 패턴을 두 upstream으로 물리 분리한다.

| 패턴 | upstream | 대상 |
|---|---|---|
| `/meetings/*/join` | `front_api_entry` | front-api 8081 (입장 전용) |
| `/meetings/*/conference-token` | `front_api_entry` | front-api 8081 |
| 그 외 | `front_api_general` | front-api 8080 |

입장 전용 8081 Connector는 `EntryConnectorConfig`가 등록하며, 단순 조회·권한 갱신과 스레드 경합을 차단한다(AS-04). 외부 서버 연계 경계는 [4.2.3.4 외부 연계](4.2.3.4-external.md)에서 다룬다.
