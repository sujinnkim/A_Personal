# 2.2.5. QA-05. 외부 서버 장애 격리

## 품질 속성 시나리오

| 항목 | 내용 |
|------|------|
| 품질속성 | 가용성 (Availability - Fault Isolation) |
| 우선순위 | 상 |
| 자극 출처 | 외부 벤더 서버 (WC서버 / VC서버 / AC서버 / Copilot Admin 서버 등) |
| 자극 | 포털 서버와 연동 중인 특정 외부 서버가 장애 또는 전면 응답 불가 상태가 됨 |
| 환경 | 정상 운영 중, 특정 외부 서버 장애 발생 (read timeout 3,000ms 이내 응답 불가) |
| 응답 | 장애 외부 서버를 사용하는 기능만 실패하고, 무관한 핵심 기능(회의 입장, 권한 갱신 등)은 정상 응답을 유지함 |
| 측정 기준 | 특정 외부 서버 장애 시 해당 서버와 무관한 핵심 기능 요청 성공률 **99.9% 이상** 유지 |

---

## 시나리오 상세

### 현행 문제

- 포털 서버는 외부 서버 호출 시 Feign 동기 호출(read timeout 3,000ms)을 사용한다.
- 외부 서버 장애 발생 시 해당 호출 스레드가 timeout 만료(3,000ms)까지 점유 상태를 유지한다.
- VC + AC 서버를 순차 호출하는 경우 최대 6,000ms 스레드가 고정될 수 있다.
- 동시에 다수 요청이 집중되면 스레드 풀이 고갈되고, 장애 외부 서버와 무관한 회의 입장·권한 갱신 등 모든 기능이 함께 응답 불가 상태가 된다.
- 외부 서버 장애를 감지하여 호출을 조기 차단하거나 우회하는 구조(서킷 브레이커, fail-fast 등)가 없다.

### 개선 목표

- 특정 외부 서버 장애가 해당 서버를 사용하는 기능 범위 내에서만 영향을 미치도록 격리한다.
- 장애 외부 서버와 무관한 핵심 기능은 스레드 풀 고갈 없이 정상 응답을 유지한다.
- 외부 서버 장애를 빠르게 감지하고 스레드를 즉시 반환하여 자원 점유를 최소화한다.

### 장애 시나리오별 격리 기준

| 외부 서버 장애 | 영향 범위 (격리 목표) | 유지되어야 할 기능 |
|-------------|-------------------|----------------|
| WC서버 장애 | 회의 입장(UC-04), 회의 시작(UC-03) 일부 | 권한 갱신(UC-01), 참석자 초대(UC-06) |
| AC서버 장애 | AC 포함 회의 시작(UC-03), AC 권한 갱신 | WC 전용 회의 입장·시작, 참석자 초대 |
| Copilot Admin 서버 장애 | LLM·용어사전 권한 갱신 (DB 폴백으로 처리) | 회의 입장·시작 등 나머지 전체 |

### 측정 방법

| 측정 항목 | 기준값 | 측정 방법 |
|---------|------|---------|
| 장애 외부 서버 무관 기능 성공률 | 99.9% 이상 | 특정 외부 서버 Mock 장애 유도 후 무관 기능 API 호출 성공 여부 확인 |
| 스레드 풀 고갈 여부 | 고갈 없음 | 장애 유도 중 스레드 풀 사용률 모니터링 |
| 장애 감지 후 복구 응답시간 | 정상 응답시간 이내 | 서킷 브레이커 오픈 이후 fallback 응답시간 측정 |
