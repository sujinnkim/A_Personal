# 2.2.1. QA-01. 로그인 권한 갱신 응답 성능

## 품질 속성 시나리오

| 항목 | 내용 |
|------|------|
| 품질속성 | 성능 (응답시간, Performance - Response Time) |
| 우선순위 | 상 |
| 자극 출처 | 미팅 사용자 (피크 시간대 동시 로그인) |

## 시나리오 상세

### 현행 문제

- 로그인 후 `GET /members/{email}` 호출 시, AC서버 / Copilot Admin 서버(LLM 권한) / Copilot Admin 서버(용어사전 권한)에 비동기 병렬 호출 후 `CompletableFuture.allOf()`로 모든 응답을 대기한다.
- 가장 느린 외부 서버의 응답 시간이 전체 응답 시간을 결정한다.
- 캐시 없이 매 로그인 요청마다 외부 서버를 호출하므로, 피크 시간대 동시 로그인이 집중될수록 외부 서버 부하가 함께 증가하여 응답 지연이 심화된다.
- 향후 연계 서버 증가 시 구조적으로 응답 시간이 악화된다.

### 개선 목표

- 피크 시간대 `GET /members/{email}` API의 평균 응답시간 1초 이내를 달성한다.
- 외부 서버 부하가 증가하는 요청 집중 구간에서도 응답 지연이 사용자 경험에 영향을 미치지 않도록 한다.

### 측정 방법

| 측정 항목 | 기준값 | 측정 방법 |
|---------|------|---------|
| 평균 응답시간 | 1초 이내 | 부하 테스트 (JMeter/k6) — 피크 부하 시나리오 수행 |
