## 시나리오 상세

### 현행 문제

- 로그인 후 `GET /members/{email}` 호출 시, AC서버 / Copilot Admin 서버(LLM 권한) / Copilot Admin 서버(용어사전 권한)에 비동기 병렬 호출 후 `CompletableFuture.allOf()`로 모든 응답을 대기한다.
- 가장 느린 외부 서버의 응답 시간이 전체 응답 시간을 결정한다.
- 피크 시간대에는 외부 서버 자체도 부하가 증가하여 개별 응답 시간이 증가하는 경향이 있다.
- 향후 연계 서버 증가 시 구조적으로 응답 시간이 악화된다.

### 개선 목표

- 권한 데이터를 Redis 캐시에 저장하여, 캐시 HIT 시 외부 서버 호출 없이 즉시 응답한다.
- 권한 변경 이벤트가 발생한 경우에만 캐시를 무효화하고 외부 서버에서 재조회한다.
- 캐시 MISS 발생 시(초기 로그인 또는 캐시 만료)에도 타임아웃 제어 하에 외부 서버를 호출하고, 부분 실패 시 기존 캐시 값을 유지한다.

### 측정 방법

| 측정 항목 | 기준값 | 측정 방법 |
|---------|------|---------|
| 평균 응답시간 | 1초 이내 | 부하 테스트 (JMeter/k6) — 피크 부하 시나리오 수행 |
| 캐시 HIT 응답시간 | 100ms 이내 | 캐시 HIT 경로 단독 측정 |
| 캐시 MISS 응답시간 | 3초 이내 (외부 서버 타임아웃 내) | 캐시 MISS 강제 시나리오 측정 |
