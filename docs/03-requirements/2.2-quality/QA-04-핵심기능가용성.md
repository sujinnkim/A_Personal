# 2.2.4. QA-04. 핵심 기능 가용성

## 품질 속성 시나리오

| 항목 | 내용 |
|------|------|
| 품질속성 | 가용성 (Availability - Success Rate) |
| 우선순위 | 상 |
| 자극 출처 | 미팅 사용자 전체, 연계시스템A/B/C |
| 자극 | 트래픽 집중 구간(피크 시간대, 스트리밍 서비스 시작 시점 등)에서 회의 입장, 회의 조회, 회의 개설 등 핵심 기능에 대한 요청이 집중됨 |
| 환경 | 피크 트래픽 집중 구간, 일부 외부 서버 응답 지연 가능 |
| 응답 | 시스템이 핵심 기능 요청에 대해 정상 응답을 반환하거나, 일부 비필수 기능의 실패를 graceful하게 처리하여 핵심 기능의 가용성을 유지함 |
| 측정 기준 | 트래픽 집중 구간 핵심 기능(회의 입장, 회의 조회, 회의 개설) 요청 성공률 **99.9% 이상** |

## 시나리오 상세

### 핵심 기능 정의

| 기능 | API | 성공률 목표 |
|------|-----|---------|
| 회의 입장 | GET /meetings/{meetingId}/join | 99.9% 이상 |
| 회의 조회 | GET /meetings, GET /meetings/{meetingId} | 99.9% 이상 |
| 회의 개설 | POST /meetings | 99.9% 이상 |

### 가용성 저해 요인

아래 요인들이 복합적으로 작용할 때 핵심 기능 성공률이 저하될 수 있다.

| 요인 | 관련 이슈 | 영향 범위 |
|------|---------|---------|
| 스레드 고갈 | ISSUE-02 | 신규 요청 처리 불가 |
| DB 커넥션 풀 고갈 | ISSUE-02, ISSUE-03 | DB 접근 필요 모든 기능 |
| 외부 서버 응답 지연 연쇄 | ISSUE-04 | 회의 개설 기능 |
| 권한 갱신 응답 지연 | ISSUE-01 | 로그인 사용자 경험 |

### 개선 전략별 가용성 기여

| 설계전략 | 가용성 기여 방식 |
|---------|--------------|
| DS-01 권한 캐시 + 이벤트 기반 갱신 | 외부 서버 의존성 제거로 권한 조회 안정성 향상 |
| DS-02 비동기 논블로킹 처리 | 스레드 고갈 방지로 동시 처리 수용량 향상 |
| DS-03 커넥션 풀 격리 (벌크헤드) | 장애 전파 차단으로 타 서비스 정상 운영 보장 |
| DS-04 로컬 캐시 + 타임아웃 세분화 | 외부 서버 지연 차단 및 fallback으로 개설 안정성 향상 |

### 측정 방법

| 측정 항목 | 기준값 | 측정 방법 |
|---------|------|---------|
| 핵심 기능 성공률 | 99.9% 이상 | 부하 테스트 (피크 시나리오) — HTTP 2xx 응답 비율 |
| 장애 격리 후 성공률 | 99.9% 이상 | 입장 풀 고갈 유도 후 조회/개설 API 성공률 측정 |
| Circuit Breaker 동작 후 성공률 | 99.9% 이상 | 외부 서버 장애 시뮬레이션 후 핵심 기능 응답 확인 |
