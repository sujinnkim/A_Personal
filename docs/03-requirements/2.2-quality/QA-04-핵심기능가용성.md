# 2.2.4. QA-04. 핵심 기능 가용성

## 품질 속성 시나리오

| 항목 | 내용 |
|------|------|
| 품질속성 | 가용성 (Availability - Success Rate) |
| 우선순위 | 상 |
| 자극 출처 | 미팅 사용자 전체 |
| 자극 | 트래픽 집중 구간(피크 시간대, 스트리밍 서비스 시작 시점 등)에서 회의 입장, 회의 시작 등 핵심 기능에 대한 요청이 집중됨 |
| 환경 | 피크 트래픽 집중 구간, 일부 외부 서버 응답 지연 가능 |
| 응답 | 시스템이 핵심 기능 요청에 대해 정상 응답을 반환하거나, 일부 비필수 기능의 실패를 graceful하게 처리하여 핵심 기능의 가용성을 유지함 |
| 측정 기준 | 트래픽 집중 구간 핵심 기능(회의 입장, 회의 시작) 요청 성공률 **99.9% 이상** |

## 시나리오 상세

### 핵심 기능 정의

| 기능 | API | 성공률 목표 |
|------|-----|---------|
| 회의 입장 | GET /meetings/{meetingId}/join | 99.9% 이상 |
| 회의 시작 | POST /meetings | 99.9% 이상 |

### 가용성 저해 요인

아래 요인들이 복합적으로 작용할 때 핵심 기능 성공률이 저하될 수 있다.

| 요인 | 관련 이슈 | 영향 범위 |
|------|---------|---------|
| 스레드/커넥션 풀 고갈 | ISSUE-01 | 신규 요청 처리 불가 |
| DB 커넥션 풀 미격리 장애 전파 | ISSUE-04 | DB 접근 필요 모든 기능 |
| 외부 서버 동기 호출 연쇄 지연 | ISSUE-05 | 회의 시작 기능 |
| 외부 벤더 서버 장애 격리 구조 부재 | ISSUE-06 | 포털 전체 가용성 |
| 권한 갱신 응답 지연 | ISSUE-02 | 로그인 사용자 경험 |
| 회의 입장·조회 처리 우선순위 미분리 | ISSUE-03 | 핵심 기능 처리 지연 |
| 입장·피드백 동시 유입 시 DB lock 경합 | ISSUE-07 | 회의 입장·조회 latency 동시 증가 |
| 단일 코드베이스로 인한 연계별 격리·정책 조정 불가 | ISSUE-08 | 피크 대응 전략 실효성 저하 |
| 선제 대응 구조 부재로 인한 cold start | ISSUE-09 | 피크 초입 핵심 기능 처리 지연 |

### 측정 방법

| 측정 항목 | 기준값 | 측정 방법 |
|---------|------|---------|
| 핵심 기능 성공률 | 99.9% 이상 | 부하 테스트 (피크 시나리오) — HTTP 2xx 응답 비율 |
| 장애 격리 후 성공률 | 99.9% 이상 | 입장 풀 고갈 유도 후 회의 시작 API 성공률 측정 |
