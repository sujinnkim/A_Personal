# UC-02. 회의 예약

## 개요

사용자가 미래 시점의 회의를 사전 예약한다. 저장된 예약 정보(시작 시각, 참석자 수)는
배치 시스템의 자동 회의 시작 트리거 기반이 되며, 피크 트래픽 예측 및 선제 대응의
근거 데이터로 활용된다.

## 행위자

- **주 행위자:** 미팅 웹포탈, 미팅 모바일 앱

## 사전 조건

- 회의 개설 권한을 보유한 사용자가 로그인된 상태이다.

## 사후 조건

- 예약 정보가 DB에 저장된다.
- 배치 시스템이 예약 시각에 회의 시작(UC-03)을 자동 트리거할 수 있는 상태가 된다.

## 기본 흐름

1. 사용자가 회의 제목, 시작 시각, 참석자, 회의 타입(WC/VC/AC) 등 예약 정보를 입력한다.
2. 포털 서버가 입력값의 유효성을 검사한다.
3. 예약 정보가 DB에 저장된다.
4. 예약 완료 안내가 클라이언트에 반환된다.

## 예외 흐름

### 시간대 중복

1. 예약 시각이 기존 예약 또는 진행 중인 회의와 충돌한다.
2. 포털 서버가 충돌 안내와 함께 예약 재입력을 유도한다.

### 권한 없음

1. 회의 개설 권한이 없는 사용자가 예약을 시도한다.
2. 포털 서버가 권한 오류를 반환한다.
