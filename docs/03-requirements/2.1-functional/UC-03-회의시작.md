# UC-03. 회의 시작

## 개요

사용자가 즉시 회의를 시작하거나, 배치 시스템이 예약 시각에 자동으로 회의를 시작한다.
front-api가 미팅 타입 결정·권한 확인·유효성 검사를 수행한 후 Meeting Manager를 거쳐
server-api가 WC/VC/AC 외부 서버에 회의 생성을 요청한다. 회의 시작 시점에 사용자
권한 갱신(UC-01)이 수행된다.

## 행위자

- **주 행위자:** 미팅 웹포탈, 미팅 모바일 앱, 배치 시스템

## 사전 조건

- 회의 개설 권한을 보유한 사용자가 로그인된 상태이다.
- 배치 트리거의 경우, 회의 예약(UC-02)이 완료된 상태이다.

## 사후 조건

- 회의가 생성되어 외부 서버(WC/VC/AC)에 등록된다.
- 참석자 초대 알림 발송(UC-10)이 수행된다.

## 기본 흐름

1. 사용자 또는 배치 시스템이 회의 시작을 요청한다.
2. 포털 서버가 회의 타입을 결정하고 사용자의 권한(SDC)을 확인한다.
3. 사용자 권한 갱신(UC-01)이 수행된다.
4. front-api가 참석자 유효성 검사 및 전처리를 수행한 후 Meeting Manager에 회의
   생성을 요청한다.
5. **WC 전용 회의**: Meeting Manager 레벨에서 회의 생성이 완결된다.
6. **VC 또는 AC 포함 회의**: server-api를 거쳐 해당 벤더 서버(VC서버 또는 AC서버)에
   Feign 동기 호출로 회의를 생성한다. 외부 연동 실패 시 전체 회의 생성이 실패한다.
7. 회의 생성이 완료되고 참석자 초대 알림 발송(UC-10)이 수행된다.

## 예외 흐름

### 외부 벤더 서버 응답 지연

1. VC서버 또는 AC서버의 Feign 동기 호출이 지연된다.
2. read timeout(3,000ms)까지 server-api 스레드가 점유 상태로 대기한다.
3. VC + AC 순차 호출 시 최대 6,000ms 스레드가 고정될 수 있다.
4. timeout 만료 후 전체 회의 생성 요청이 실패로 반환된다.

### 외부 벤더 서버 장애

1. VC서버 또는 AC서버가 오류를 반환한다.
2. 장애 격리 구조가 없어 전체 회의 생성 요청이 실패한다.

### 권한 없음

1. 회의 개설 권한이 없는 사용자가 시작을 요청한다.
2. 포털 서버가 권한 오류를 반환한다.
