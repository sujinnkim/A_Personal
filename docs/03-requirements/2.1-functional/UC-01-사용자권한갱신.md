# UC-01. 사용자 권한 갱신

## 개요

로그인 완료 후 클라이언트가 자동으로 권한 갱신을 요청한다. 포털 서버가 AC서버(AC 권한)와
Copilot Admin 서버(LLM 권한·용어사전 권한)에 비동기 병렬 호출을 수행하고, 모든 응답
수신 후 갱신된 권한 정보를 클라이언트에 반환한다.

## 행위자

- **주 행위자:** 미팅 웹포탈, 미팅 모바일 앱

## 사전 조건

- 로그인이 완료되어 액세스 토큰이 발급된 상태이다.

## 사후 조건

- AC 권한, LLM 권한, 용어사전 권한이 갱신되어 클라이언트에 반환된다.
- 갱신된 권한 정보가 DB에 저장된다.

## 기본 흐름

1. 클라이언트가 로그인 완료 직후 자동으로 회원 정보 조회(`GET /members/{email}?type=detail`)를 요청한다.
2. 포털 서버가 AC서버, Copilot Admin 서버(LLM 권한), Copilot Admin 서버(용어사전 권한)에
   비동기 병렬 호출을 시작한다.
3. 포털 서버가 모든 외부 서버의 응답을 수신한다.
4. 포털 서버가 갱신된 권한 정보를 DB에 저장한다.
5. 포털 서버가 권한 정보를 포함한 회원 정보를 클라이언트에 반환한다.
6. 클라이언트가 응답을 수신하고 사용자에게 서비스 화면을 표시한다.

## 예외 흐름

### 외부 서버 응답 지연

1. 병렬 호출 대상 중 하나 이상이 응답을 지연한다.
2. 포털 서버는 가장 늦게 응답하는 서버를 기다리므로 전체 응답시간이 지연된다.
3. read timeout(3,000ms) 만료 시 해당 요청이 실패하고 권한 갱신 전체가 실패 응답으로 반환된다.

### 외부 서버 장애

1. 외부 서버 중 하나 이상이 오류를 반환하거나 연결에 실패한다.
2. 장애 격리 구조가 없어 해당 서버의 오류가 전체 권한 갱신 요청 실패로 전파된다.
3. 클라이언트는 오류 응답을 수신하고, 사용자는 로그인 후 서비스 화면에 진입하지 못한다.
