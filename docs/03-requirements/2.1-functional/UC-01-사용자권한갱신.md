# UC-01. 사용자 권한 갱신

## 개요

로그인 완료 후 클라이언트가 자동으로 권한 갱신을 요청한다. 포털 서버가 AC서버(AC 권한)와
Copilot Admin 서버(LLM 권한·용어사전 권한)에 비동기 병렬 호출을 수행하고, 모든 응답
수신 후 갱신된 권한 정보를 클라이언트에 반환한다.

## 행위자

- **주 행위자:** 미팅 웹포탈, 미팅 모바일 앱

## 사전 조건

- 로그인이 완료되어 액세스 토큰이 발급된 상태이다.

## 사후 조건

- AC 권한, LLM 권한, 용어사전 권한이 반환된다.
- 정상 갱신된 권한은 DB에 저장되고, 외부 서버 오류·지연으로 갱신에 실패한 권한은
  DB에 저장된 기존 값이 반환된다.

## 기본 흐름

1. 클라이언트가 로그인 완료 직후 자동으로 회원 정보 조회(`GET /members/{email}?type=detail`)를 요청한다.
2. 포털 서버가 AC서버, Copilot Admin 서버(LLM 권한), Copilot Admin 서버(용어사전 권한)에
   비동기 병렬 호출을 시작한다.
3. 포털 서버가 모든 외부 서버의 응답을 수신한다.
4. 포털 서버가 갱신된 권한 정보를 DB에 저장한다.
5. 포털 서버가 권한 정보를 포함한 회원 정보를 클라이언트에 반환한다.
6. 클라이언트가 응답을 수신하고 사용자에게 서비스 화면을 표시한다.

## 예외 흐름

### 외부 서버 응답 지연 (timeout)

1. 병렬 호출 대상 중 하나 이상이 read timeout(3,000ms) 이내에 응답하지 않는다.
2. 포털 서버는 timeout이 발생한 서버의 권한을 DB에 저장된 기존 값으로 대체하여 반환한다.
   전체 요청은 성공 응답으로 완료되지만 반환된 해당 권한은 현재 실제 권한과 다를 수 있다.
3. **권한이 실제보다 넓게 반환된 경우**: 해당 외부 서버에 실제 요청이 도달하면 권한
   없음 오류가 발생한다.
4. **권한이 실제보다 좁게 반환된 경우**: 사용자가 실제로 사용 가능한 기능을 이용하지
   못하는 상태가 된다.

### 외부 서버 오류

1. 외부 서버 중 하나 이상이 오류를 반환하거나 연결에 실패한다.
2. 포털 서버는 오류가 발생한 서버의 권한을 DB에 저장된 기존 값으로 대체하여 반환한다.
   나머지 서버의 권한은 정상적으로 갱신된다. 전체 요청은 성공 응답으로 완료되지만
   오류 서버에 해당하는 권한은 현재 실제 권한과 다를 수 있다.
3. 권한 불일치로 인한 영향은 응답 지연(timeout) 케이스와 동일하다.
