# UC-09. 회의록 생성

## 개요

회의 종료(UC-08) 후 시스템이 meeting-ai 서버를 통해 종료된 회의의 참석자·발언·채팅
등 데이터를 수집하고 회의록을 생성한다. 사용자가 직접 트리거하는 흐름이 아닌 시스템
자동 실행이다.

## 행위자

- **주 행위자:** 시스템 (회의 종료 후 자동 실행)

## 사전 조건

- 회의 종료(UC-08)가 완료된 상태이다.

## 사후 조건

- 회의록이 생성되어 저장된다.

## 기본 흐름

1. 회의 종료 후 시스템이 해당 회의의 데이터(참석자, 발언, 채팅 등)를 수집한다.
2. 포털 서버가 meeting-ai 서버에 회의록 생성을 요청한다.
3. meeting-ai 서버가 분석 결과를 반환한다.
4. 생성된 회의록이 저장된다.

## 예외 흐름

### meeting-ai 서버 장애

1. meeting-ai 서버가 오류를 반환하거나 응답하지 않는다.
2. 회의록 생성이 실패하며, 이후 재시도가 가능한 상태로 남는다.

### 회의 데이터 부족

1. 발언 또는 채팅 데이터가 충분하지 않아 회의록 생성이 불가하다.
2. 빈 회의록 또는 안내 메시지가 저장된다.
