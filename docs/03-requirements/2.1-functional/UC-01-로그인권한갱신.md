# 2.1.3.1. UC-01. 로그인 및 권한 갱신

## 유스케이스 개요

| 항목 | 내용 |
|------|------|
| Use Case ID | UC-01 |
| Use Case명 | 로그인 및 권한 갱신 |
| 관련 기능 요구사항 | FR-01 |
| 관련 이슈 | ISSUE-02 |
| 행위자 | 미팅 사용자, 미팅 웹, 미팅 모바일 앱 |
| 사전조건 | 미팅 웹/앱 접속 완료, 사용자 계정 보유 |
| 사후조건 | 로그인 완료 (액세스 토큰 발급), 권한 정보(AC 권한, LLM 권한, 용어사전 권한) 갱신 및 반환됨 |

---

## 기본 흐름

| 단계 | 행위자 | 설명 |
|------|--------|------|
| 1 | 미팅 사용자 | 이메일/비밀번호를 입력하고 로그인 요청 |
| 2 | 미팅 웹 | `POST /sign-in` 요청 전송 |
| 3 | Authentication Service | 자격증명 검증 후 JWT 액세스 토큰 발급 및 반환 |
| 4 | 미팅 웹 | 액세스 토큰 수신 후 자동으로 `GET /members/{email}?type=detail` 호출 |
| 5 | Meeting Portal 서버 | AC서버, Copilot Admin 서버(LLM 권한), Copilot Admin 서버(용어사전 권한)에 `CompletableFuture`를 사용하여 비동기 병렬 호출 시작 |
| 6 | Meeting Portal 서버 | `CompletableFuture.allOf()`로 모든 외부 서버 응답을 동기 대기 (가장 느린 서버의 응답시간이 전체 응답시간 결정) |
| 7 | AC서버 | AC 권한 갱신 및 조회 결과 반환 (Feign 동기 호출, read timeout 3,000ms) |
| 7 | Copilot Admin 서버 | LLM 권한 갱신 및 조회 결과 반환 (Feign 동기 호출, read timeout 3,000ms) |
| 7 | Copilot Admin 서버 | 용어사전 권한 갱신 및 조회 결과 반환 (Feign 동기 호출, read timeout 3,000ms) |
| 8 | Meeting Portal 서버 | 갱신된 권한 정보를 DB에 저장 |
| 9 | Meeting Portal 서버 | 회원 정보 및 권한 정보를 포함한 응답 반환 |
| 10 | 미팅 웹 | 응답 수신 후 사용자에게 서비스 화면 표시 |

---

## 대안 흐름

### A1. 외부 서버 응답 지연

| 단계 | 행위자 | 설명 |
|------|--------|------|
| 6a | Meeting Portal 서버 | 외부 서버(AC서버 또는 Copilot Admin 서버) 중 하나 이상이 read timeout(3,000ms) 이내에 응답하지 않음 |
| 7a | Meeting Portal 서버 | `CompletableFuture.allOf()` 대기 중 전체 응답이 지연됨. 가장 늦게 응답하는 서버가 전체 응답시간을 결정함 |
| 8a | Meeting Portal 서버 | 타임아웃 만료 후 예외 처리 → 전체 요청 실패(5xx) 반환 |

### A2. 외부 서버 오류 (5xx, 연결 실패)

| 단계 | 행위자 | 설명 |
|------|--------|------|
| 7b | Meeting Portal 서버 | 외부 서버 중 하나 이상이 오류를 반환하거나 연결에 실패함 |
| 8b | Meeting Portal 서버 | 장애 격리 구조 없음. 외부 서버 오류가 전체 `GET /members/{email}` 요청 실패로 전파됨 |
| 9b | 미팅 웹 | 5xx 응답 수신. 권한 갱신 실패로 인해 로그인 후 서비스 화면 진입 불가 |

### A3. 로그인 자격증명 오류

| 단계 | 행위자 | 설명 |
|------|--------|------|
| 3a | Authentication Service | 자격증명 불일치 → 401 Unauthorized 반환 |
| 4a | 미팅 웹 | 오류 메시지 표시, `GET /members/{email}` 호출 없음 |

| API | HTTP 메서드 | 경로 | 설명 |
|-----|-----------|------|------|
| 로그인 | POST | /sign-in | 자격증명 검증 및 JWT 발급 |

## 아키텍처 관련 주목 사항

이 유스케이스는 ISSUE-02의 핵심 발현 경로다.

- `GET /members/{email}?type=detail` 호출 시 AC서버, Copilot Admin 서버(LLM), Copilot Admin 서버(용어사전) 등 다수의 외부 서버에 비동기 병렬 호출을 수행한다. 그러나 `CompletableFuture.allOf()`로 모든 응답을 동기 대기하므로, 비동기 병렬 호출임에도 불구하고 가장 느린 외부 서버의 응답시간이 전체 응답시간을 결정한다.
- 서버사이드 캐시가 없어 AC 권한·LLM 권한·용어사전 권한은 변경 빈도가 낮음에도 매 로그인마다 모든 외부 서버에 갱신을 요청한다. 업무 시작 시간대(오전 9시, 오후 1시)에 동시 로그인이 집중되면 외부 서버에 대한 요청도 동시에 폭증하여 외부 서버 응답 지연이 가중된다.
- 외부 서버 장애 격리 구조(Circuit Breaker 등)가 없어 외부 서버 1개 오류가 전체 권한 갱신 요청 실패로 전파된다.
- 향후 연계 서버 추가 계획이 있어, 현재 `allOf()` 대기 구조에서 연계 서버 증가에 비례하여 구조적 지연이 심화될 위험이 있다.
