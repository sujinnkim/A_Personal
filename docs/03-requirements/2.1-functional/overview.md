# 2.1. 기능 요구사항

본 절은 미팅 포털 서버 아키텍처 개선 설계의 대상이 되는 기능 요구사항을 정의한다. 각 요구사항은 유스케이스로 상세화되며, 식별된 이슈(→ 1.2 이슈 참조)와 연결된다.

## 2.1.1. 기능 요구사항 목록

| ID | 요구사항명 | 설명 | 우선순위 | 관련 유스케이스 | 관련 이슈 |
|----|-----------|------|---------|--------------|---------|
| FR-01 | 로그인 시 권한 갱신 | 로그인 완료 후 AC 권한, LLM 권한, 용어사전 권한 등을 갱신하고 반환한다. 피크 시간대에도 안정적으로 처리되어야 한다. | 상 | UC-01 | ISSUE-01 |
| FR-02 | 회의 입장 처리 | 입장 파라미터(OTP 포함)를 생성하여 응답한다. 2만 명 동시 입장 burst 상황에서도 안정적으로 처리되어야 한다. | 상 | UC-03 | ISSUE-02 |
| FR-03 | 회의 개설 | 외부 서버(WC/VC/AC) 호출을 포함한 회의 개설을 처리한다. 동기 호출 구간을 최소화하여 응답 시간을 단축한다. | 상 | UC-02 | ISSUE-04 |
| FR-04 | 회의 조회 | 피크 시간대 집중 조회를 효율적으로 처리한다. DB 커넥션 고갈 없이 안정적인 응답을 제공한다. | 중 | UC-04 | ISSUE-03 |

## 2.1.2. 외부 연계 시스템 참조

기능 요구사항 수행에 관련된 외부 연계 시스템은 다음과 같다.

| 시스템 | 역할 | 관련 기능 요구사항 |
|--------|------|----------------|
| WC서버 (약 10개 지역) | 웹 컨퍼런스 개설, OTP 발급 | FR-02, FR-03 |
| VC서버 (약 3개 지역) | 비디오 컨퍼런스 개설, 회의실 참석 처리 | FR-03 |
| AC서버 | 오디오 컨퍼런스 개설, 전화 참석 처리, AC 권한 갱신 | FR-01, FR-03 |
| AI서버 | AI 기능 연계 | FR-03 |
| Copilot Admin 서버 | LLM 권한 갱신, 용어사전 권한 갱신 | FR-01 |
| 연계시스템A | 동시 1000건 회의 개설, 1분간 10만 건 조회, 1분간 2만 건 참석 | FR-02, FR-03, FR-04 |
| 연계시스템B | 회의실 장비 특정 시간 점유 | FR-03 |
| 연계시스템C | 일반 통합회의 연계 개설 | FR-03 |
