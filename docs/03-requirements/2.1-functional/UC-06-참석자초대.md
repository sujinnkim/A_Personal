# UC-06. 참석자 초대

## 개요

진행 예정 또는 진행 중인 회의에 참석자를 추가 초대한다. 외부 서버(WC/VC/AC)에
참석자 정보를 업데이트하고, 초대 알림 발송(UC-10)이 수행된다.

## 행위자

- **주 행위자:** 미팅 웹포탈, 미팅 모바일 앱

## 사전 조건

- 초대 권한을 보유한 사용자가 로그인된 상태이다.
- 초대 대상 회의가 존재하는 상태이다.

## 사후 조건

- 참석자 정보가 DB 및 외부 서버에 등록된다.
- 초대 알림 발송(UC-10)이 수행된다.

## 기본 흐름

1. 사용자가 초대할 참석자를 검색하고 선택한다.
2. 포털 서버가 외부 서버(WC/VC/AC)에 참석자 정보를 업데이트한다.
3. 참석자 정보가 DB에 저장된다.
4. 초대 알림 발송(UC-10)이 수행된다.

## 예외 흐름

### 이미 참석 중인 사용자

1. 이미 회의에 참석 중이거나 초대된 사용자를 중복 초대한다.
2. 포털 서버가 중복 안내 메시지를 반환한다.

### 외부 서버 오류

1. 벤더 서버(WC/VC/AC)에 참석자 등록 요청이 실패한다.
2. 장애 격리 구조가 없어 참석자 초대 전체가 실패로 반환된다.
