# UC-04. 회의 입장

## 개요

사용자가 예약 상태 또는 진행 중인 회의에 입장한다. 포털 서버가 DB에서 입장 가능
여부를 먼저 확인한 후 conference-token을 발급하고 입장 파라미터를 생성하여 WC서버에
Feign 동기 호출(read timeout 3,000ms)로 전달한다. 입장 파라미터 반환 후 인미팅
클라이언트(PC 또는 모바일)가 실행된다.

## 행위자

- **주 행위자:** 미팅 웹포탈, 미팅 모바일 앱

## 사전 조건

- 입장 대상 회의가 예약 상태 또는 진행 중인 상태이다.

## 사후 조건

- 입장 파라미터가 클라이언트에 반환된다.
- 인미팅 클라이언트(PC 클라이언트 또는 모바일 클라이언트)가 실행되어 사용자가
  회의에 참여한 상태가 된다.

## 기본 흐름

1. 사용자가 회의 번호, 회의 URL, 또는 참석자 목록에서 입장을 요청한다.
2. 포털 서버가 DB에서 입장 가능 여부를 확인한다.
   - 사용자가 해당 회의의 참석자로 초대되어 있는지, 또는
   - 회의 번호·URL로 누구나 참석 가능한 오픈 회의인지 확인한다.
3. 확인이 통과되면 포털 서버가 conference-token을 발급한다.
4. 포털 서버가 사용자 권한 정보를 포함한 입장 파라미터를 생성한다.
5. 포털 서버가 WC서버에 입장 파라미터를 Feign 동기 호출로 전달하고 응답을 수신한다.
6. 포털 서버가 입장 파라미터를 클라이언트에 반환한다.
7. 인미팅 클라이언트(PC 클라이언트 또는 모바일 클라이언트)가 실행되어 회의에 참여한다.

## 예외 흐름

### 입장 불가 (참석자 미초대, 오픈 회의 아님)

1. DB 확인 결과 사용자가 참석자로 초대되어 있지 않고, 오픈 회의도 아니다.
2. 포털 서버가 입장 불가 오류를 반환한다.
3. conference-token이 발급되지 않는다.

### WC서버 응답 지연

1. WC서버의 Feign 동기 호출 응답이 지연된다.
2. 해당 요청의 스레드가 최대 3,000ms 점유 상태로 대기한다.
3. 2만 명 동시 입장 시 대량의 스레드가 동시에 점유 상태가 되어 스레드 풀 고갈
   위험이 있다.
4. read timeout 만료 시 입장 요청이 실패로 반환된다.

### DB 커넥션 풀 고갈

1. 동시 입장 집중으로 DB 커넥션 풀이 고갈된다.
2. 입장 가능 여부 확인이 불가능해져 입장 처리가 실패한다.
3. 단일 커넥션 풀 공유 구조에서 다른 기능(참석자 조회, 참석자 초대 등)도 함께
   영향을 받는다.
