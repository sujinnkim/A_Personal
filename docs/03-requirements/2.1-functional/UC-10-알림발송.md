# UC-10. 회의 초대/참석 알림 발송

## 개요

회의 시작(UC-03) 또는 참석자 초대(UC-06) 시 in서버(SMS), 메일 서버, push 서버를 통해
관련 알림을 발송한다. 알림 발송 실패가 회의 시작·참석자 초대 등 핵심 기능에 영향을
주지 않도록 처리된다.

## 행위자

- **주 행위자:** 시스템 (UC-03 또는 UC-06에 «include»로 자동 실행)

## 사전 조건

- 알림 발송 대상 참석자 목록이 존재한다.
- UC-03(회의 시작) 또는 UC-06(참석자 초대)가 실행 중이다.

## 사후 조건

- 알림 발송 요청이 in서버(SMS), 메일 서버, push 서버에 전달된다.
- 발송 결과와 무관하게 상위 UC(UC-03 또는 UC-06)의 흐름이 정상 완료된다.

## 기본 흐름

1. 상위 UC(UC-03 또는 UC-06)가 알림 발송 대상 목록을 전달한다.
2. 포털 서버가 in서버(SMS), 메일 서버, push 서버에 발송을 요청한다.
3. 발송 요청 전달이 완료된다.

## 예외 흐름

### 특정 채널 발송 실패

1. in서버(SMS), 메일 서버, push 서버 중 하나 이상에서 발송이 실패한다.
2. 해당 채널의 실패는 다른 채널 발송 및 상위 UC의 정상 흐름에 영향을 주지 않는다.
3. 실패 내역이 기록된다.
