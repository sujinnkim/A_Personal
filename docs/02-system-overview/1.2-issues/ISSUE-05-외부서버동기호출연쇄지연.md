# ISSUE-05. 외부 서버 동기 호출 연쇄 지연

## 현황

회의 개설 API는 사용자 요청에 대한 응답을 반환하기 전에 WC서버, VC서버, AC서버 등 외부 서버에 순차적으로 동기 호출을 수행한다. 별도의 타임아웃 설정이나 장애 차단 메커니즘 없이 외부 서버의 응답을 그대로 대기하므로, 정상 범위 내의 응답 지연(예: 200ms~800ms)도 포털 서버 응답에 그대로 반영된다.

```
회의 개설 요청
  → WC서버 호출 (동기) → 응답 대기
  → VC서버 호출 (동기) → 응답 대기
  → AC서버 호출 (동기) → 응답 대기
  → 전체 응답 반환
```

## 문제점

- 외부 서버(WC/VC/AC)의 응답 지연이 포털 서버의 회의 개설 응답 지연으로 직접 연쇄된다.
- 3개 서버 순차 호출 시 각 서버의 지연이 누적된다.
- 피크 시간대에는 외부 서버 자체도 부하 증가로 인해 응답 지연이 증가하는 경향이 있다.
- 자주 변경되지 않는 설정 정보(회의 설정, 권한 정책 등)도 매 요청마다 외부 서버를 호출한다.
- 외부 서버 중 하나가 느려지거나 장애가 발생하면 해당 호출 스레드가 타임아웃까지 점유 상태로 유지된다.

## 영향

- 회의 개설 응답 시간 증가 (→ QA-01 위반 위험)
- 외부 서버 중 하나의 지연이 전체 개설 흐름을 지연시키는 구조
- 피크 시간대 회의 개설 성공률 저하 위험 (→ QA-04 위반 위험)
