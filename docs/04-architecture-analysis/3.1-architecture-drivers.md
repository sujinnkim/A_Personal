# 3.1. 아키텍처 드라이버

아키텍처 드라이버는 아키텍처 설계 결정에 직접적인 영향을 미치는 핵심 요소다. 품질 요구사항(QA), 기능 요구사항(FR), 아키텍처 제약사항(CR)에서 도출되며, 중요도·난이도를 기준으로 설계 우선순위를 결정한다.

## 3.1.1. 아키텍처 드라이버 목록

|----|----------|------|------|------|:----:|:----:|

## 3.1.2. 핵심·보조 드라이버 분류

```
핵심 드라이버 (중요도·난이도 모두 상)
   AD-01 피크 시 권한 갱신 응답시간
   AD-02 8만 명 동시 입장 처리 성능
   AD-04 핵심 기능 성공률 99.9%
   AD-05 로그인 시 권한 갱신 처리 (기능)
   AD-06 8만 명 동시 회의 입장 (기능)

보조 드라이버 (중요도 상, 난이도 중)
   AD-03 DB 커넥션 풀 장애 격리
   AD-07 외부 서버 포함 회의 시작 (기능)
   AD-09 Java Spring Boot + HikariCP 기술 스택 준수 (제약)
   AD-10 기존 API 하위 호환성 유지 (제약)
```

핵심 드라이버는 미달성 시 서비스 목적이 곧바로 훼손되며 복수의 설계 전략을 복합 적용해야 해결 가능하다. 보조 드라이버는 미달성 영향이 동일하나 명확한 패턴(Bulkhead, Circuit Breaker, API 버저닝 등)으로 해결 가능하여 난이도가 상대적으로 낮다.
