# 3.1. 아키텍처 드라이버

아키텍처 드라이버는 아키텍처 설계 결정에 직접적인 영향을 미치는 핵심 요소다. 품질 요구사항(QA), 기능 요구사항(FR), 아키텍처 제약사항(CON)에서 도출되며, 우선순위를 기준으로 설계 방향을 결정한다.

## 3.1.1. 아키텍처 드라이버 목록

| ID | 드라이버명 | 유형 | 우선순위 | 출처 | 설명 |
|----|----------|------|---------|------|------|
| AD-01 | 피크 시 권한 갱신 응답시간 | QA | 상 | QA-01 | 로그인 시 `GET /members/{email}` API 평균 응답시간 1초 이내 달성. 외부 서버 allOf 대기 구조를 캐시 기반으로 전환 필요 |
| AD-02 | 2만 명 동시 입장 처리 성능 | QA | 상 | QA-02 | 2만 명 동시 입장 burst 시 DB 커넥션 풀 사용률 80% 이하, 평균 응답시간 1초 이내 달성. 논블로킹 처리 및 캐시 최적화 필요 |
| AD-03 | DB 커넥션 풀 장애 격리 | QA | 상 | QA-03 | 입장 서비스 커넥션 풀 고갈 시 타 서비스(조회, 개설) 정상 운영 보장. 기능별 커넥션 풀 분리 필요 |
| AD-04 | 핵심 기능 성공률 99.9% | QA | 상 | QA-04 | 트래픽 집중 구간 회의 입장/조회/개설 성공률 99.9% 이상. 장애 격리, 논블로킹, 캐시, 타임아웃 조합 필요 |
| AD-05 | 로그인 시 권한 갱신 처리 | 기능 | 상 | FR-01 | 매 로그인 시 AC 권한, LLM 권한, 용어사전 권한 갱신 및 반환. 피크 시에도 정상 처리 |
| AD-06 | 2만 명 동시 회의 입장 | 기능 | 상 | FR-02 | 입장 파라미터 생성 및 반환. 대규모 동시 처리 안정성 확보 |
| AD-07 | 외부 서버 포함 회의 개설 | 기능 | 상 | FR-03 | WC/VC/AC 서버 연동 포함 회의 개설. 동기 호출 구간 최소화 |
| AD-08 | 피크 시 회의 조회 처리 | 기능 | 중 | FR-04 | 피크 시간대 집중 조회의 안정적 처리. 입장 처리 burst와 격리된 독립적 처리 필요 |
| AD-09 | Java Spring Boot + HikariCP 기술 스택 준수 | 제약 | 상 | CON-01, CON-02 | 구현 기술은 Java Spring Boot 3.x, HikariCP로 제한됨. 설계 결정이 이 스택 내에서 구현 가능해야 함 |
| AD-10 | 기존 API 하위 호환성 유지 | 제약 | 상 | CON-04 | 클라이언트 변경 없이 서버 측 개선 적용. API 경로/스키마 불변. 내부 처리 방식만 변경 가능 |

## 3.1.2. 드라이버 우선순위 분류

### 핵심 드라이버 (설계 방향 결정에 필수)

| 드라이버 | 이유 |
|---------|------|
| AD-01 (권한 갱신 응답시간) | 로그인 사용자 경험에 직결. 외부 서버 의존 구조 변경이 필수 |
| AD-02 (동시 입장 처리 성능) | 2만 명 burst가 스레드/DB 고갈을 유발하는 즉각적 위험 |
| AD-03 (DB 커넥션 풀 장애 격리) | 단일 기능 장애가 전체 서비스로 전파되는 구조적 결함 |
| AD-04 (핵심 기능 성공률) | 서비스 안정성의 최종 지표, 모든 개선이 이를 향해 수렴 |
| AD-10 (API 하위 호환성) | 이를 위반하면 클라이언트 측 변경이 필요하여 적용 불가 |

### 중요 드라이버 (설계 품질에 영향)

| 드라이버 | 이유 |
|---------|------|
| AD-05, AD-06, AD-07, AD-08 | 기능 요구사항으로 시나리오 커버리지 결정 |
| AD-09 | 구현 가능성을 제한하는 기술 제약 |
