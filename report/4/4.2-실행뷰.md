### 4.2. 아키텍처 실행 설계

아키텍처 실행 설계는 설계 전략(AS)이 구조에 반영된 모습을 세 관점으로 구체화한다. 서버 측 컴포넌트 결합과 동작을 다루는 실행 뷰(4.2.1), 애플리케이션 빌드 단위의 의존을 다루는 모듈 뷰(4.2.2), 하드웨어 배치를 다루는 배치 뷰(4.2.3)이다. 실행 뷰는 전체 C&C(Component and Connector) 구조를 먼저 제시한 뒤 주요 유스케이스를 시퀀스로 확대한다.

| 목차 | 내용 | 관련 요구·전략 |
| :---: | ----- | ----- |
| 4.2.1 실행 뷰 | Overall C&C View + 주요 유스케이스 시퀀스(입장·권한·워밍·Bulkhead·CB) | QA-01·02·03·05 / AS-02~09 |
| 4.2.2 모듈 뷰 | front-api 패키지 구조·의존과 패키지별 상세(entry·auth·meeting·integration·config) | AS-01 / 헥사고날 port·adapter |
| 4.2.3 배치 뷰 | GCP·GKE 종합 배치도·이중화·네트워크·데이터·외부 연계 | AS-03·04·07·08 / RK-01·07 |
