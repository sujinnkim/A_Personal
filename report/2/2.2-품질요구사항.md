QA-01·02·04는 본 과제의 핵심 주제인 "요청 집중 구간 대응"을 직접 정량화한 결과로, 미달성 시 서비스 목적이 곧바로 훼손된다. QA-03·05는 미달성 시 영향이 동일하나 명확한 패턴(Bulkhead, Circuit Breaker)으로 해결 가능하여 난이도가 상대적으로 낮다. 모든 QA는 3.1 아키텍처 드라이버에서 직접 인용된다.

품질 요구사항은 성격에 따라 두 층으로 구분된다. **QA-04(핵심 기능 성공률)는 다른 품질 요구사항이 함께 충족될 때 달성되는 종합 결과 지표**이고, **QA-01(응답시간)·QA-02(처리량)·QA-03(내부 커넥션 풀 격리)·QA-05(외부 서버 장애 격리)는 각각 독립된 원인·메커니즘 기준**이다. 특히 QA-03과 QA-05는 각각 "내부 자원 격리"와 "외부 연계 격리"라는 서로 다른 결함 격리 축을 담당하며, QA-04의 하위 항목이 아니라 QA-04를 떠받치는 독립 요구사항이다.

위 저해 요인들은 QA-04가 단독으로 해결하는 항목이 아니라, 각각 대응하는 품질 요구사항에서 개별적으로 다루어진다. 응답 지연은 QA-01, 처리량·자원 고갈은 QA-02, 내부 커넥션 풀 격리는 QA-03, 외부 서버 장애 격리는 QA-05. QA-04는 이들이 함께 충족될 때 달성되는 핵심 기능의 **종합 성공률 지표**로서, 개별 저해 요인의 원인 기준이 아니라 결과 기준을 정의한다.

#### 2.2.5. QA-05. 외부 서버 장애 격리

QA-05는 QA-03(내부 커넥션 풀 격리)과 짝을 이루는 **외부 연계 격리 축**으로, 외부 벤더 서버 장애가 무관한 핵심 기능으로 전파되지 않도록 보장한다. QA-04(종합 성공률)의 하위 항목이 아니라, QA-04 달성을 위한 독립적 결함 격리 요구사항이라는 점에서 QA-04와 역할이 구분된다. 설계 목표 관점에서 QA-05는 DG-04(핵심 기능 가용성)를 **외부 서버 장애 격리 측면에서 구체화**한 것으로, QA-05 전용 설계 목표(DG)를 별도로 두지 않고 DG-04를 공유한다.

**품질 속성 시나리오**

| 항목 | 내용 |
| :---: | ----- |
| 품질속성 | 가용성 (Availability - Fault Isolation) |
| 우선순위 | 상 |
| 자극 출처 | 외부 벤더 서버 (WC서버 / VC서버 / AC서버 / Copilot Admin 서버 등) |
| 자극 | 포털 서버와 연동 중인 특정 외부 서버가 장애 또는 전면 응답 불가 상태가 됨 |
| 환경 | 정상 운영 중, 특정 외부 서버 장애 발생 (read timeout 3,000ms 이내 응답 불가) |
| 응답 | 장애 외부 서버를 사용하는 기능만 실패하고, 무관한 핵심 기능(회의 입장, 권한 갱신 등)은 정상 응답을 유지함 |
| 측정 기준 | 특정 외부 서버 장애 시 해당 서버와 무관한 핵심 기능 요청 성공률 **99.9% 이상** 유지 |
